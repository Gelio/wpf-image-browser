WPF 2 - część pierwsza

Image browser

  • Aplikacja przykładowa
  • Opis:
    • zadanie polega na napisaniu prostej aplikacji umożliwiającej przeglądanie folderów ze zdjęciami
    • użytkownik może otwierać cały folder lub wyświetlać pojedyncze zdjęcia
  • Wymagania:
    • Główne okno:
      • minimalny rozmiar 500 x 500, rozmiar początkowy 800 x 600
      • na górze okna pasek menu
      • pozostałą część okna wypełnia slider oraz część główna, w której wyświetlane będą miniaturki obrazków
      • napis "Image Browser" na pasku tytułowym okna
      • okno pojawia się na środku ekranu
      • kolor tła: #FFC9B1B1
    • Górne menu:
      • 'File' zawiera:
        • 'Open image' - wyświetla okienko dialogowe, umożliwiające wybór obrazka w jednym z trzech formatów (.jpg, .gif, .png) i otwiera nowe okno z wybranym obrazkiem
        • 'Open folder' - wyświetla okienko dialogowe, umożliwiające wybór folderu z obrazkami
        • 'Exit' - zamyka aplikację, oddzielony separatorem od wcześniejszych
      • 'About' - wyświetla MessageBox z krótką informacją o programie
    • Główna część programu:
      • okno podzielone na dwie części: prawą (kolor tła: #FF49A8DE) i lewą (kolor tła: #FF8b9a59)
      • w lewej części znajduje się menu z wystylizowanymi przyciskami
      • wszystkie przyciski mają eliptyczny kształt, przezroczyste brzegi oraz zmieniają wygląd przy najechaniu myszą i wciśnięciu
      • kolory użyte dla przycisków to: biały, cyjan i magenta
      • w prawej części znajduje się slider i odpowiednio wystylizowana lista z miniaturkami obrazków znajdujących się w wybranym przez nas folderze
      • slider umożliwia dynamiczną modyfikację rozmiaru miniaturek obrazków (50 - 250) - musi zostać użyte bindowanie
      • każda miniaturka zawiera podgląd obrazka, którego dotyczy, nazwę pliku oraz posiada dodany efekt cienia występujący z prawej strony i z dołu
      • rozmiar obrazka w miniaturkach może być zmieniany przy pomocy slidera od 50 do 250 px
      • podwójne kliknięcie w dowolną miniaturkę powoduje otwarcie nowego okna z obrazkiem, którego tytuł jest nazwą otwartego pliku
      • obrazek otwarty w nowym oknie powinien zachowywać oryginalne wymiary, a okno nie powinno dać się skalować
  • Uwagi:
    • aplikacja jest odporna na skalowanie okna, kontrolki dostosowują się do zmian rozmiaru okna aplikacji
    • lista z miniaturkami powinna wyświetlać scrollbar w przypadku jej przepełnienia
    • w razie niejasności program musi być zgodny z zachowaniem i wyglądem aplikacji przykładowej (z wyjątkiem potencjalnych bugów)
  • Wskazówki:
    • INotifyPropertyChanged
    • ListView, ListView.ItemTemplate, DataTemplate
    • ListView.ItemsPanel, WrapPanel
    • Border.Effect
    • FolderBrowserDialog (System.Windows.Forms)
    • ControlTemplate, TemplateBinding
    • Trigger, IsMouseOver, IsPressed
  • Punktacja:
    • wystylizowane menu po lewej stronie + poprawne otwieranie obrazków w nowym oknie + wyświetlanie zawartości wybranego folderu w dowolny sposób, który umożliwia otwarcie obrazka poprzez podwójne kliknięcie - 4 punkty
    • górne menu + wyświetlanie obrazków znajdujących się w wybranym folderze (ścieżka może być zahardcodowana) w postaci odpowiednio wystylizowanej listy miniaturek (jak w aplikacji przykładowej) + możliwość zmiany rozmiaru miniaturek przy pomocy slidera- 4 punkty

WPF 2 - część druga

  • Aplikacja przykładowa
  • Wymagania:
    • Główne okno:
      • główne okno zostało podzielona na dwie części, których szerokość można zmieniać przy pomocy splittera
      • w pierwszej części znajduje się: menu z wystylizowanymi buttonami z części laboratoryjnej, checkbox umożliwiający pokazywanie/ukrywanie drzewa partycji i katalogów, kontrolka wyświetlająca wszystkie partycje oraz pozwalająca na przeglądanie struktury katalogowej każdego dysku
      • domyślnie drzewo katalogów jest wyświetlone, a checkbox zaznaczony
      • po odznaczeniu checkboxa kontrolka z drzewem katalogów zanika (animacja po parametrze przezroczystości)
      • po ponownym zaznaczeniu checkboxa kontrolka ponownie się pojawia (ponownie animacja)
      • po kliknięciu w nazwę katalogu zostają wyświetlone miniaturki obrazków, które się w nim znajdują
      • w drugiej części wyświetlane są miniaturki obrazków
    • Okno z obrazkiem:
      • rozmiar otwieranego okna powinien być dostosowany do rozdzielczości ekranu i wymiarów obrazka, tzn. proporcje obrazka powinny być zachowane, ale okno w całości musi mieścić się na ekranie (należy wzorowac się na działaniu przykładowej aplikacji)
      • okno zostało podzielone na 3 sekcje
      • po prawej stronie znajduje się największa sekcja, w której wyświetlany jest wybrany obrazek
      • po lewej stronie okna znajdują się dwie rozwijane sekcje
      • pierwsza sekcja zawiera podstawowe informacje o obrazku, takie jak jego nazwa, wymiary i data utworzenia
      • druga sekcja pozwala na wybór z rozwijanej listy pluginu oraz jego zastosowanie do obrazka
      • każda sekcja powinna mieć inny kolor tła i obramowania
    • Menu kontekstowe:
      • wyświetla się po kliknięciu prawym przyciskiem myszy na obrazku w oknie z obrazkiem
      • menu zawiera:
        • 'Save' - pozwala na zapisanie obrazka w jednym z następujących formatów: png, jpg, bmp.
        • 'Reset' - przywraca oryginalny obrazek (do stanu przed zastosowaniem jakichkolwiek pluginów)
        • 'Exit' - zamyka okno z obrazkiem
    • System pluginów:
      • system pluginów powinien umożliwiać dodawanie nowych funkcjonalności bez potrzeby ponownej kompilacji aplikacji, poprzez wrzucenie nowej dll-ki do katalogu z plikiem exe
      • każdy plugin powinien być zaimplementowany jako biblioteka (kompilowana do pliku dll) z klasą implementującą odpowiedni interfejs
      • interfejs systemu pluginów powinien zostać zaimplementowany w sposób umożliwiający łatwe udostęponienie go innym programistom
      • należy zaimplementować przynajmniej dwa pluginy pozwalające na obracanie obrazka (obroty o 90 i 180 stopni)
      • poniżej interfejs, który został użyty w aplikacji przykładowej i którego należy użyć przy implementacji zadania, a także sposób implementacji przykładowego pluginu: 
        public interface IPlugin
{
    string Name { get; }
    BitmapImage Do(BitmapImage image);
}
                            
        public class ExamplePlugin : IPlugin
{
    public string Name
    {
        get
        {
            return "Name of Example Plugin";
        }
    }

    public BitmapImage Do(BitmapImage bmpOriginal)
    {
        var bmp = DoSomethingWithImage(bmpOriginal);

        return bmp;
    }
}
      • poprawnie zaimplementowany system pluginów powinien umożliwiać wymianę dll-kami i ich poprawne działanie
  • Wskazówki:
    • TreeView, TreeViewItem
    • GridSplitter
    • Environment.GetLogicalDrives(), Directory.GetDirectories(), DirectoryInfo
    • DoubleAnimation, Opacity
    • Expander
    • Image.ContextMenu
    • Label.ContentStringFormat
    • BitmapImage
    • SystemParameters.PrimaryScreenWidth, SystemParameters.PrimaryScreenHeight
    • Build Path w Project properties (przyspiesza pisanie i testowanie kodu)
  • Punktacja:
    • implementacja drzewa katalogów - 3 punkty
    • możliwość zmiany szerokości kontrolek w głównym oknie - 0.5 punkta
    • poprawne dostosowywanie rozmiarów nowego okna z obrazkiem - 1 punkt
    • wyświetlanie informacji o obrazku w rowijanym oknie - 1 punkt
    • implementacja menu kontekstowego - 1 punkt
    • animowane pokazywanie i ukrywanie kontrolki z drzewem katalogów w zależności od zaznaczenia checkboxa - 1.5 punkta
    • implementacja systemu pluginów (bez systemu po 0.5 punkta za plugin) - 4 punkty
